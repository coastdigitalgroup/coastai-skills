# Referral Loop Audit Checklist

Use this checklist to evaluate and optimize the referral mechanics of any website or application.

## 1. Timing & Placement (The Trigger)
- [ ] **Joy Peak identified:** Is the referral prompt triggered after a moment of high value (success, achievement, delight)?
- [ ] **Visibility:** Is the referral program easy to find without being intrusive?
- [ ] **Frequency:** Are users prompted at reasonable intervals (not every session)?
- [ ] **Contextual:** Does the prompt relate to the action the user just took?

## 2. Incentive Structure
- [ ] **Two-sided:** Do both the Referrer and the Referee receive a benefit?
- [ ] **Alignment:** Does the reward align with why people use the product (e.g., storage for Dropbox, credit for Uber)?
- [ ] **Clarity:** Is the value of the reward immediately obvious (e.g., "Get $20" vs "Earn points")?
- [ ] **Accessibility:** Is the reward easy to earn (not buried behind too many requirements)?

## 3. The Sharing Experience (Reducing Friction)
- [ ] **One-click sharing:** Are there direct buttons for WhatsApp, Slack, Email, etc.?
- [ ] **Native API:** Does the mobile site use the Web Share API for native OS sharing?
- [ ] **Pre-filled copy:** Is there a default message that sounds natural and helpful?
- [ ] **Personal link:** Is the referral link short, branded, and easy to copy?

## 4. The Recipient Experience (The Landing Page)
- [ ] **Personalization:** Does the landing page mention the friend who sent the invite?
- [ ] **Benefit-first:** Is the referral reward (e.g., "You have $20 waiting") the most prominent element?
- [ ] **Social Proof:** Does the page reinforce why their friend is using the product?
- [ ] **Low-friction signup:** Is the path from landing to "referred signup" as short as possible?

## 5. Tracking & Transparency
- [ ] **Status Dashboard:** Can the referrer see the status of their invites (Sent, Pending, Successful)?
- [ ] **Notification:** Is the referrer notified immediately when a reward is earned?
- [ ] **Fraud Prevention:** Are there basic checks in place to prevent self-referral abuse?
