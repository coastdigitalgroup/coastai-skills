recommendation engine.

## 2. The Question Flow Audit
*Review your proposed or existing questions against these friction-reduction
rules.*

| Question | Purpose (Goal, Filter, or Personalizer?) | Can a 10-year-old answer this? |
| :--- | :--- | :--- |
| 1. | | [ ] Yes |
| 2. | | [ ] Yes |
| 3. | | [ ] Yes |
| 4. | | [ ] Yes |

**Checklist:**
- [ ] First question is "easy-in" (low cognitive load).
- [ ] No more than 7 questions total.
- [ ] Progress bar is visible.
- [ ] Images/Icons are used for at least 50% of the answers.

## 3. Results Page Strategy
*Ensure the transition from "Learner" to "Buyer" is seamless.*

- **The "Why" Statement:** How will you explain the match?
  *(Example: "Based on your [Answer X] and [Answer Y], we recommend...")*
  - Draft: ____________________________________________________________________
- **The Primary CTA:**
  - [ ] Direct "Add to Cart" button.
  - [ ] "Start Free Trial" button.
  - [ ] Benefit-driven label (e.g., "Get My Custom Plan").
- **Secondary Options:**
  - [ ] "Why not this?" (Briefly explain why the #2 option wasn't #1).
  - [ ] "Email my results" (Optional, non-blocking).

## 4. Friction & Technical Audit
- [ ] **Mobile Check:** Are all buttons at least 44px tall?
- [ ] **Zero-Results Check:** If no perfect match exists, what is the fallback?
- [ ] **Lead Gen Check:** Is the email gate *after* the results or clearly
  optional?
- [ ] **Loading State:** Is there a "Calculating your results..." animation to
  build perceived value (The Labor Illusion)?
