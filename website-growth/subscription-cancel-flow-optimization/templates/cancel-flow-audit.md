# Subscription Cancel Flow Audit Checklist

Use this template to audit an existing subscription cancellation process and identify opportunities for retention and recovery.

## 1. Accessibility & Compliance
- [ ] **Visibility:** Is the "Cancel" option easy to find in the account/billing settings (not hidden in deep sub-menus)?
- [ ] **Method Parity:** Can users cancel using the same method they used to sign up (e.g., if they signed up online, can they cancel online)?
- [ ] **Compliance:** Does the flow meet regional requirements (e.g., California's "Online Cancel" or EU's "Cancel Button")?

## 2. Feedback & Segmentation
- [ ] **Exit Survey:** Does the flow ask for the reason for cancellation?
- [ ] **Actionable Data:** Are the survey options specific enough to trigger different recovery paths (e.g., distinguishing between "Buggy" and "Too Hard to Use")?
- [ ] **Open Feedback:** Is there an optional text box for users to provide more detail?

## 3. Recovery & "Save" Offers
- [ ] **Pause Option:** Is a "Pause" (30-90 days) offered as an alternative to cancellation?
- [ ] **Contextual Discount:** Is a retention discount offered specifically to those who select "Price/Value" as their reason?
- [ ] **Maintenance Plan:** For data-heavy apps, is there a low-cost "Keep my data" tier?
- [ ] **Relevance:** Does the offer directly address the user's stated reason for leaving?

## 4. Persuasion & Loss Aversion
- [ ] **Value Reminder:** Does the flow remind the user of the benefits/data they will lose?
- [ ] **Streak/Progress:** Does it highlight personal progress or "streaks" that will be reset?
- [ ] **Grandfathered Status:** If the user is on an old, cheaper price, is this pointed out?

## 5. UX & Final Confirmation
- [ ] **Click Count:** Can the entire flow (including survey and offer) be completed in 3 clicks or fewer?
- [ ] **Clear Status:** Is it 100% clear when the cancellation is finalized (e.g., a "Success" page)?
- [ ] **Access Clarity:** Does the user know exactly how much time they have left in their current period?
- [ ] **Mobile Friendly:** Are buttons large, distinct, and easy to tap on mobile?

## 6. Post-Cancellation
- [ ] **Confirmation Email:** Is an automated confirmation sent immediately?
- [ ] **Reactivation Path:** Is there a "One-Click Reactivate" link in the confirmation email/page?
- [ ] **Nurture Pivot:** Is the user moved from a "Customer" email list to a "Former Customer" nurture list?
