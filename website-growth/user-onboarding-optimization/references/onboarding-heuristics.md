# Onboarding Heuristics & Principles

Optimizing onboarding is as much about psychology as it is about UI design. Use these principles to guide your strategy.

## 1. The "Aha! Moment"

The moment a user internalizes the value of your product.

- **Rule:** Every second spent before the "Aha! Moment" is a second where the user might churn.
- **Example:** For a photo app, it's not "Creating an account"—it's "Seeing your first photo filtered and shared."

## 2. Time-to-Value (TTV)

The time it takes for a new user to reach the "Aha! Moment."

- **Goal:** Minimize TTV. Remove every click, form field, and loading screen that stands between the user and their goal.

## 3. The Zeigarnik Effect

The psychological tendency to remember uncompleted tasks better than completed ones.

- **Application:** Use onboarding checklists with a "pre-completed" first step (e.g., "Account Created ✓") to create a desire to "finish the set."

## 4. The Endowed Progress Effect

People are more likely to complete a goal if they feel they have already made progress toward it.

- **Application:** Start a 5-step progress bar at 20% complete by counting the "Sign Up" as the first step.

## 5. Progressive Disclosure

A design pattern that moves from the simple to the complex.

- **Application:** Only show the user the features they need *right now*. Don't overwhelm a new user with "Settings" and "Integrations" on their first visit.

## 6. The "Empty State" Opportunity

A screen with no data is a blank canvas for instruction.

- **Rule:** Never show a blank screen without a "Primary Action."
- **Good:** "You have no contacts yet. [Import from Google]"
- **Bad:** "No contacts found."

## 7. Cognitive Load & Choice Paralysis

The more choices you give a user, the less likely they are to make any choice at all.

- **Application:** During onboarding, give the user ONE choice per screen. If you must have multiple options, highlight the "Recommended" path.
