# Interactive Demo Audit Checklist

Use this checklist to evaluate and optimize any self-guided product tour or
interactive demo.

## 1. Value Alignment (The "Aha!" Moment)
- [ ] Does the demo start within 1 click of a high-value "Aha!" moment?
- [ ] Is the primary pain point of the persona addressed in the first 3 steps?
- [ ] Have "setup" steps (logins, settings, configs) been removed or automated?

## 2. Narrative & Pacing
- [ ] Is the total length under 10 steps?
- [ ] Does every step require an interaction (click, type, hover) rather than
      just a "Next" button?
- [ ] Is there a clear "story" being told (Problem -> Solution -> Outcome)?
- [ ] Are there "skip" options for users who want to jump to the end?

## 3. Microcopy & Tooltips
- [ ] Is every tooltip benefit-driven ("Save time...") rather than feature-
      driven ("This button...")?
- [ ] Does the copy tell the user *why* they are clicking?
- [ ] Is the language free of internal technical jargon?
- [ ] Is there a clear directive in every bubble (e.g., "Click the Blue Button")?

## 4. UI & Visuals
- [ ] Is the demo responsive? (Test on 13" laptop vs. 27" monitor).
- [ ] Are tooltips positioned so they don't cover critical UI elements?
- [ ] Does the "Start" state feel inviting rather than overwhelming?
- [ ] Is the "Exit" button clearly visible if a user gets stuck?

## 5. Conversion Strategy
- [ ] Is there a primary CTA on the final screen?
- [ ] Does the final CTA match the value shown in the demo?
- [ ] If the demo is gated, is the gate placed *after* a value realization?
- [ ] Is there a "secondary" action (e.g., "Share this tour") for users not
      ready to buy?

## 6. Technical Performance
- [ ] Does the demo load in under 3 seconds?
- [ ] Are all animations smooth and non-distracting?
- [ ] Does the demo work on the most common browsers (Chrome, Safari, Edge)?
