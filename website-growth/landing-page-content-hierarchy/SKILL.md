# Landing Page Content Hierarchy

## Purpose

The Landing Page Content Hierarchy skill provides a systematic framework for
organizing the "body" of a landing page. While the Hero section grabs attention
and the Form captures the lead, the hierarchy of the content in between must
logically move the user from initial interest to a state of high intent. This
skill focuses on strategic sequencing, messaging prioritization, and visual
scannability to reduce cognitive load and improve conversion rates.

## Use Cases

- Optimizing a long-form sales page for a complex product.
- Improving the "mid-page" engagement of a SaaS landing page.
- Re-structuring a page where heatmaps show users scroll but don't click.
- Aligning a landing page's flow with a specific user journey or persona.

## When NOT to Use

- **Simple "Splash" Pages:** Where the only content is a hero and a button.
- **Purely Informational Articles:** Blog posts or documentation where the goal
  is reading, not conversion.
- **Deep Application Interfaces:** Dashboards or settings pages where functional
  utility outweighs persuasive flow.
- **E-commerce Product Detail Pages (PDPs):** Which often follow a rigid,
  template-driven hierarchy better served by PDP-specific optimization.

## Inputs

1. **Target Audience Profile:** Who is the user and what is their "awareness
   level" (Unaware, Problem Aware, Solution Aware)?
2. **Current Page Structure:** A list of existing sections and their order.
3. **Primary Goal/CTA:** What is the specific action we want the user to take at
   the end of the flow?
4. **Core Value Prop & Key Benefits:** The primary reasons why a user should
   choose this solution.
5. **Common Objections:** Known reasons why users hesitate to convert.

## Outputs

1. **Content Hierarchy Audit:** Identification of logical gaps, repetitive
   sections, or misplaced information.
2. **Revised Structural Map:** A recommended sequence of sections based on
   conversion frameworks.
3. **Section-Level Messaging Guidance:** Specific focus areas for each section
   (e.g., "Section 3 should transition from the Problem to the Solution").
4. **Scannability Recommendations:** Guidance on headers, subheaders, and visual
   cues to guide the eye.

## Workflow

### 1. Inventory and Awareness Assessment

List all current sections. Determine the user's starting awareness level.

- If **Problem Aware:** Start with the pain point (Agitation).
- If **Solution Aware:** Start with the unique advantage (Differentiation).

### 2. Select a Structural Framework

Choose a logical sequence for the content. Common patterns include:

- **Problem-Agitation-Solution:** Define the problem, explain why it's painful,
  then present the solution.
- **Attention-Interest-Desire-Action:** Move from broad benefit to specific
  features to emotional desire.
- **The "High-Intent" Flow:** Lead with the solution, follow with proof, then
  handle objections.

### 3. Prioritize Messaging

Map the key benefits to the sections.

- **Primary Benefit:** Must appear in the first 30% of the page.
- **Proof & Evidence:** Must follow closely after any major claim.
- **Objection Handling:** Should be placed near the final CTA or where the user
  is likely to hesitate.

### 4. Optimize for Scannability

Ensure a user can understand the core value by reading ONLY the headlines.

- Use **Descriptive Headlines:** Instead of "Our Features," use "Save 10 Hours a
  Week with Automated Reporting."
- Use the **Z-Pattern or F-Pattern:** Align key information where eyes naturally
  land.
- **Visual Breaks:** Use images, icons, or background color shifts to signal a
  change in topic.

### 5. Review Against Decision Rules

Verify the new flow against the growth heuristics below.

## Decision Rules

- **The Narrative Thread:** Every section must logically lead to the next. If
  you can move a section to a different part of the page without losing context,
  the hierarchy is likely weak.
- **Hierarchy of Needs:** Answer "What is it?" before "How does it work?" and
  "How does it work?" before "Why should I trust you?"
- **Scannable Value:** If a user "skims" the page in 10 seconds, they should
  still understand the primary benefit and the CTA.
- **The "One Idea" Rule:** Each section or block of content should communicate
  exactly ONE key idea or benefit.

## Constraints

- **Content Accuracy:** All claims, statistics, and proof points used throughout the page hierarchy must be factually accurate.
- **Brand Voice:** Reordering and rewriting sections must preserve the established brand voice and messaging framework.
- **Technical Dependency:** Section reordering may require developer or CMS involvement depending on the page builder or template in use.

## Non-Goals

- Writing new copy or designing new visual elements for sections.
- Technical implementation of the page layout or CMS configuration.
- SEO keyword strategy or on-page SEO optimization.

## Common Failure Patterns

- **The "Feature Dump":** Listing every technical detail before establishing the
  value or problem.
- **The "Logical Leap":** Asking for the sale (CTA) before providing enough
  information or proof to justify it.
- **Repetitive Messaging:** Having multiple sections that say the same thing in
  slightly different ways, causing "scroll fatigue."
- **Burying the Lead:** Hiding the most important benefit at the bottom of the
  page.
- **Uniform Visual Weight:** Making every section look identical, which prevents
  the user from identifying the most important information.

## Validation Criteria

- [ ] **The "Headlines Only" Test:** Read only the H2s on the page. Does it tell a
  complete and persuasive story?
- [ ] **Scroll Mapping:** Analyze where users stop scrolling. A sharp drop-off
  usually indicates a "boring" or "illogical" section.
- [ ] **Squint Test:** Squint at the page. Is the logical progression of sections
  and the primary CTA still clear?
- [ ] **User Sequence Survey:** Ask testers to rank the sections in order of
  "importance to their decision." If their ranking differs from your layout,
  adjust the hierarchy.
