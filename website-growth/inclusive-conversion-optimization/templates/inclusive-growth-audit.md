# Template: Inclusive Growth Audit Checklist

Use this checklist to identify "Inclusive Friction" in your primary conversion
funnel. Scoring "No" on any of these items indicates a gap in your reachable
market share.

## 1. Perceivability (The "Can I see/hear it?" Test)

- [ ] **Contrast:** Do all CTAs and critical text have a contrast ratio of at
  least 4.5:1 against their background? (Test in "High Glare" simulation).
- [ ] **Text Size:** Is the primary body text at least 16px, and can it be zoomed
  to 200% without breaking the layout?
- [ ] **Sensory Redundancy:** Is information conveyed through more than just
  color? (e.g., Error icons + color, underlined links).
- [ ] **Media Clarity:** Does all video content have captions and a text
  summary?

## 2. Operability (The "Can I use it?" Test)

- [ ] **Keyboard Path:** Can you navigate from the Home page to a successful
  Checkout/Signup using ONLY the `Tab`, `Enter`, and `Space` keys?
- [ ] **Focus Visibility:** Is it obvious which element currently has focus at
  all times? (No "invisible" focus rings).
- [ ] **Touch Targets:** Are all mobile buttons and links at least 44x44px to
  prevent "fat-finger" errors in high-distraction environments?
- [ ] **No Traps:** Can a user exit every modal or dropdown using the `Esc`
  key?

## 3. Understandability (The "Does it make sense?" Test)

- [ ] **Persistent Labels:** Do all form fields have labels that remain visible
  while the user is typing?
- [ ] **Helpful Errors:** Do error messages explain *how* to fix the problem,
  rather than just stating that an error occurred?
- [ ] **Progress Indicators:** In multi-step flows, is it clear where the user
  is and how much work remains?
- [ ] **Plain Language:** Is the value proposition and CTA free of internal
  jargon or complex metaphors?

## 4. Robustness (The "Does it stay working?" Test)

- [ ] **Hardware Agnostic:** Does the conversion flow work on a 5-year-old
  smartphone or a low-resolution tablet?
- [ ] **Connectivity Tolerance:** Does the page remain functional (e.g., buttons
  don't break) if certain assets (like custom fonts) fail to load?
- [ ] **State Persistence:** If a user is interrupted and returns 5 minutes
  later, is their form progress saved?
