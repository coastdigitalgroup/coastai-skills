# High-Performance 404 Page Breakdown

This example demonstrates how to transform a "dead-end" 404 error into a
discovery opportunity.

## The Problem
A standard 404 page often says "404 Not Found" and provides no way out. This
results in immediate user abandonment (bounce).

## The Solution
A high-performance 404 page uses the `error-and-status-system` to provide
context, empathy, and clear navigation paths.

### Visual Layout
1. **The Visual Hook (Top):** A playful illustration of a "lost" character or
   a broken compass. This reduces user anxiety.
2. **The Human Headline:** "We can't find that page." (Large H1)
3. **The Empathy Description:** "The link might be broken, or the page may have
   moved. Don't worry, we'll help you find what you're looking for."
4. **Primary Recovery (Center):** A prominent Search bar with a placeholder
   "What are you looking for?"
5. **Secondary Discovery (Bottom):** A grid of "Popular Destinations":
   - Our Latest Blog Posts
   - Product Catalog
   - Help Center
   - Contact Support
6. **The Safety Net (Footer-adjacent):** A "Back to Homepage" primary button.

## Outcome
- **Reduced Bounce Rate:** Users continue their journey instead of closing the
  tab.
- **Brand Trust:** The professional handling of a failure signals a
  high-quality service.
- **Findability:** The search bar allows the user to correct the "broken link"
  immediately.

# 500 Server Error: Reassurance Pattern

When the server fails, the goal shifts from discovery to reassurance.

### Anatomy of a 500 Page
- **State Indicator:** A "System Maintenance" or "Service Interruption" icon.
- **Headline:** "Something went wrong on our end."
- **Action:** A "Try Refreshing the Page" button (primary) and "Check System
  Status" link (secondary).
- **Subordinated Metadata:** A small, monospaced string at the bottom:
  `Reference ID: 9f82-a312-7721`. This helps support agents find the logs
  without bothering the user with technical details.
