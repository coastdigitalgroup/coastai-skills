# Integrations Directory Audit & Optimization Checklist

Use this template to audit your existing integrations directory and identify high-impact opportunities to improve findability, clarity, and user activation.

## 1. Findability & Discovery
- [ ] **Search Prominence:** Is there a search bar? Is it the first thing users see on desktop and mobile?
- [ ] **Site-Wide Search:** Do your integrations appear in the global website/app search results?
- [ ] **Category Logic:** Are integrations grouped by user goals (e.g., "Marketing Automation") rather than technical types?
- [ ] **SEO Landing Pages:** Does every major integration have its own indexable URL?
- [ ] **"Request" Fallback:** Is there a clear way for users to request an integration that doesn't exist yet?

## 2. Grid & List View (The Gallery)
- [ ] **Logo Clarity:** Are high-resolution, recognizable logos used for every partner?
- [ ] **Scannable Descriptions:** Does each card explain *what* the integration does in 10 words or less?
- [ ] **Capability Badges:** Are "Sync Types" (Two-way, Trigger, Action) clearly labeled on the card?
- [ ] **Social Proof:** Are usage stats ("Used by 1k+ teams") or star ratings visible?
- [ ] **Primary Action:** Does the card link to a setup page rather than just a generic help article?

## 3. Integration Detail Page (IDP)
- [ ] **Outcome-Focused Headline:** Does the H1 focus on the *benefit* of connecting (e.g., "Automate your lead sync")?
- [ ] **Use Case Clarity:** Are at least 3 specific business use cases listed?
- [ ] **Friction Preview:** Is there a screenshot of the configuration screen or a "Setup Time" estimate?
- [ ] **Direct Activation:** If the user is logged in, does the CTA lead directly to the integration setup in-app?
- [ ] **"Missing" Alternative:** If the integration is limited, do you suggest a workaround (e.g., Zapier, Make, or Webhooks)?

## 4. Technical & Trust Signals
- [ ] **Author Identification:** Is it clear if the integration was built by "Your Company," the "Partner," or a "Third Party"?
- [ ] **Maintenance Status:** Are "Beta" or "Legacy" integrations clearly marked?
- [ ] **Security/Compliance:** Are there badges for "Certified," "Secure," or "SOC2 Compliant" where applicable?
- [ ] **Version History:** Is there a date or version number to show the integration is actively maintained?

## 5. Mobile & Interaction
- [ ] **Responsive Grid:** Does the directory transition to a 1 or 2-column list on mobile with large tap targets?
- [ ] **Sticky Search:** On mobile, does the search/filter bar remain accessible as the user scrolls?
- [ ] **Instant Feedback:** Does the "Connect" button provide immediate visual feedback (e.g., loading state)?
