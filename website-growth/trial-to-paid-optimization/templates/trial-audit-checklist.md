# Trial-to-Paid Conversion Audit Checklist

Use this checklist to identify friction points in your trial-to-paid journey and uncover opportunities for contextual upgrades.

## 1. Transparency & Visibility
- [ ] **Trial Status Indicator:** Is the user's current trial status (days/usage remaining) visible within the app interface at all times?
- [ ] **Expiration Warnings:** Are there clear notifications sent at logical intervals (e.g., 7 days, 3 days, 24 hours) before the trial ends?
- [ ] **Feature Labeling:** Are "Paid" or "Pro" features clearly distinguished from "Free" features within the UI (e.g., using a crown icon or badge)?

## 2. Upgrade Triggers
- [ ] **Contextual Prompts:** Does an upgrade prompt appear immediately when a user clicks on or attempts to use a gated "Pro" feature?
- [ ] **Usage Milestones:** Do you trigger upgrade reminders when a user reaches 80% or 90% of their free usage limit?
- [ ] **Value Reinforcement:** Do your upgrade prompts mention the specific benefit the user was just trying to access?

## 3. The Paywall Experience
- [ ] **Value Recap:** Does the upgrade screen summarize the value the user has already received (e.g., "You've created 5 reports")?
- [ ] **Plan Comparison:** Is there a clear "Free vs. Paid" comparison table that justifies the price jump?
- [ ] **Frictionless Form:** If the user already provided their name/email during signup, is that data pre-filled in the upgrade/payment form?
- [ ] **Risk Reversal:** Are trust signals (SSL, "Cancel Anytime," "Secure Checkout") placed directly next to the "Pay" button?

## 4. Retention & Recovery
- [ ] **The "Soft Landing":** If the trial expires, does the user have the option to downgrade to a limited "Free" version instead of being completely locked out?
- [ ] **Extension Offer:** Do you offer a one-time trial extension (e.g., +7 days) for high-intent users who haven't converted yet?
- [ ] **Exit Survey:** Do you ask users *why* they chose not to upgrade at the moment of expiration?

## 5. Mobile Considerations
- [ ] **One-Tap Upgrade:** On mobile, do you support Apple Pay/Google Pay to reduce payment friction?
- [ ] **Thumb-Friendly:** Is the "Upgrade" button in the "Natural Thumb Zone" at the bottom of the screen?
