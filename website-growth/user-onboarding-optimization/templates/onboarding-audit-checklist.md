# User Onboarding Audit Checklist

Use this template to evaluate the "First Mile" experience of any website or application.

## 1. Friction & Flow Audit

- [ ] **Signup Length:** Does the signup require more than 3-4 fields? (Can any be delayed?)
- [ ] **Verification Wall:** Does the flow stop for email/phone verification before the user sees the value?
- [ ] **Social Auth:** Are Google/Apple/SSO options provided to reduce typing?
- [ ] **Breadcrumbs/Progress:** If multi-step, does the user know how many steps are left?

## 2. The Path to "Aha!"

- [ ] **Clear Definition:** Is the "Aha! Moment" defined for this product?
- [ ] **Direct Path:** Can the user reach that moment in under 2 minutes?
- [ ] **No Distractions:** Are there any popups or "tours" that distract from the primary setup goal?
- [ ] **Sensible Defaults:** Are fields pre-filled where possible to reduce decision fatigue?

## 3. Interface & Guidance

- [ ] **The "Empty State":** Does every blank screen provide a clear "Call to Action" to add data?
- [ ] **Progressive Disclosure:** Are advanced features hidden until the basics are mastered?
- [ ] **Onboarding Checklist:** Is there a persistent way for users to see their progress (e.g., a checklist)?
- [ ] **Skip" Option:** Can users skip the tutorial or setup wizard if they are already familiar?

## 4. Retention & Success Signals

- [ ] **Immediate Reward:** Is there a "Success" message or celebration when the first task is finished?
- [ ] **Next Steps:** Once the "Aha!" moment is reached, is the *next* milestone clear?
- [ ] **Email Follow-up:** Is there a "Day 1" email that reinforces the value and encourages the next step?
