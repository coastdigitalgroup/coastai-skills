# Psychological Persistence Triggers

Optimizing for returning visitors is as much about psychology as it is about
data. Use these three core triggers to create a "sticky" multi-session
experience.

## 1. The Zeigarnik Effect (The "Unfinished Task" Itch)
**The Principle:** People remember uncompleted or interrupted tasks better than
completed ones. An unfinished task creates "psychic tension" that is only
released upon completion.

**How to Apply:**
- **Progress Bars:** Show a progress bar for an anonymous checkout or signup
  that the user started but didn't finish.
- **"Resume" Prompts:** Use language like "You were almost there" or "Finish
  setting up your profile" to trigger the need for closure.

## 2. The Endowment Effect (The "Already Mine" Bias)
**The Principle:** People ascribe more value to things merely because they own
them or feel a sense of possession over them.

**How to Apply:**
- **"Your" vs. "The":** Use "Your Cart" or "Your Saved Items" instead of "Guest
  Cart."
- **Pre-filled State:** If a user has made selections (e.g., color, size, or
  plan tier), keep those selections active. When they return, the product
  already "reflects" their choices, making it feel like it already belongs in
  their world.

## 3. Recognition over Recall (The "Path of Least Resistance")
**The Principle:** It is much easier for the human brain to recognize something
from a list than to recall it from memory.

**How to Apply:**
- **Recently Viewed:** Don't force users to remember the name of the product
  they liked; show it to them.
- **Recent Searches:** Surface the last 3 search terms.
- **Smart Defaults:** If the user previously selected "Small Business" in a
  persona-selector, default the site content to that segment on their return.

## 4. The Sunk Cost Fallacy (Light Version)
**The Principle:** Users are more likely to continue an endeavor if they have
already invested time, effort, or data into it.

**How to Apply:**
- **Form Persistence:** By saving partial form data, you respect the "effort"
  the user already put in. Deleting that data "punishes" their previous
  investment and makes them less likely to start over.
