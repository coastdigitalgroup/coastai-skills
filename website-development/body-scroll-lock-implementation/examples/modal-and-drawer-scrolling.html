Body Scroll Lock Demonstration

This interactive demonstration compares a Standard Desktop Scroll Lock against a Robust Cross-Platform (iOS Safari-safe) Scroll Lock.

Standard Lock (Fails on iOS)

Implementation A: Standard overflow: hidden

This implementation sets overflow: hidden on the <body> element when the modal is active.

The iOS Bug: If you open this modal on an iPhone/iPad or inside a mobile responsive emulator with touch simulation, tapping the background backdrop or swiping on the modal header will still cause the underlying page content to scroll up and down. This results in the background shifting and breaking layout continuity.

Robust Lock (Works on iOS)

Implementation B: Core iOS Touch Interception

This implementation applies overflow: hidden on desktop, but activates a touchmove event coordinator for iOS touch users.

The iOS Solution: It catches touchmove events on the document and prevents them, except inside marked container elements. Inside these scrollable elements, it measures finger movement (direction) to block scrolls at the extreme boundaries (fully scrolled up or down), preventing the scroll from leaking to the background <body>.
